Debug and fix an AMBA APB-based RTL design.

**Context**:
You are given a buggy SystemVerilog RTL implementation of an APB system consisting of:

- apb_master
- apb_slave
- apb_top

The design currently compiles but contains multiple functional and protocol-related bugs.

Your Task

1. Fix the RTL design in the `sources/` directory
2. Create additional testcases in the testbench
3. Add assertions to validate APB protocol behavior

RTL Fix Requirements

1. apb_top
- Instantiate apb_master and apb_slave
- Properly connect APB signals:
  - PADDR, PWDATA, PRDATA
  - PWRITE, PSEL, PENABLE
  - PREADY, PSLVERR

2. apb_master
Implement a correct APB FSM:

- IDLE:
  - PSEL = 0, PENABLE = 0
  - Wait for TRANSFER

- SETUP:
  - PSEL = 1, PENABLE = 0
  - Drive address and control signals

- ACCESS:
  - PSEL = 1, PENABLE = 1
  - Wait for PREADY
  - Return to IDLE

Ensure:
- No skipping of SETUP phase
- Stable signals during ACCESS
- Proper reset behavior

 3. apb_slave
- Implement register memory
- Use word-aligned addressing:
  - index = PADDR[ADDR_WIDTH-1:2]

- Write:
  - PSEL=1 & PENABLE=1 & PWRITE=1

- Read:
  - PSEL=1 & PENABLE=1 & PWRITE=0

- Reset:
  - Registers initialized to 0

- Outputs:
  - PREADY = 1
  - PSLVERR = 0

Assertions Requirement

Add assertions (SystemVerilog or cocotb checks) to verify:

- PSEL must be high during SETUP and ACCESS
- PENABLE must only be high in ACCESS phase
- Read data is valid when PREADY is high

Verification

The design will be tested using cocotb tests in `tests/`.

Expected checks:
- Functional correctness
- Protocol behavior (basic level)

Goal
Produce a correct and stable APB RTL design and enhance verification with meaningful testcases and assertions.
