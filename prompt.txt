##================================================================================
##PROBLEM: APB Master–Slave RTL Debug and Integration
##================================================================================
Debug and fix an AMBA APB-based RTL design.

**Context**:
You are given a buggy SystemVerilog RTL implementation of an APB system consisting of:

- apb_master
- apb_slave
- apb_top

The design currently compiles but contains multiple functional and protocol-related bugs.

Known issues include:
- apb_top does NOT instantiate or connect the master and slave
- apb_master has incorrect FSM transitions and violates APB protocol timing
- apb_slave has incorrect memory behavior, unstable PREADY, and improper reset handling

---

**Your Task**:

Fix the RTL design in the `sources/` directory so that it behaves as a correct APB system.

---

### Required Fixes

#### 1. apb_top
- Instantiate both `apb_master` and `apb_slave`
- Properly connect all APB interface signals:
  - PADDR, PWDATA, PRDATA
  - PWRITE, PSEL, PENABLE
  - PREADY, PSLVERR
- Ensure signal directions and widths match correctly

---

#### 2. apb_master
Fix the FSM to correctly follow APB protocol:

States:
- IDLE
- SETUP
- ACCESS

Correct behavior:

- IDLE:
  - PSEL = 0, PENABLE = 0
  - On TRANSFER → move to SETUP

- SETUP:
  - PSEL = 1, PENABLE = 0
  - Drive address, write control, and write data
  - Move to ACCESS in next cycle

- ACCESS:
  - PSEL = 1, PENABLE = 1
  - Wait for PREADY = 1
  - On completion → return to IDLE

Additional requirements:
- Do NOT skip SETUP phase
- Maintain stable signals during ACCESS
- Properly reset all outputs

---

#### 3. apb_slave
Fix memory and protocol behavior:

- Implement a register file of size REG_NUM
- Use proper address decoding:
  - index = PADDR[ADDR_WIDTH-1:2] (word aligned)

- Write Operation:
  - When PSEL=1, PENABLE=1, PWRITE=1 → write to memory

- Read Operation:
  - When PSEL=1, PENABLE=1, PWRITE=0 → output PRDATA

- Reset behavior:
  - All registers must reset to 0

- Fix outputs:
  - PREADY must always be 1 (no toggling)
  - PSLVERR must always be 0

---

### Important Constraints

- Do NOT change module interfaces (ports)
- Do NOT add new signals unless necessary
- Keep design synthesizable
- Maintain clean FSM coding style
- Avoid latches

---
See docs/Specification.md for detailed APB protocol behavior and signal descriptions.
